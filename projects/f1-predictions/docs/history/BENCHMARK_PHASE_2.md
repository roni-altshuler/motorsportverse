# Benchmark Phase 2 — Elite Heads Break the Podium-Hit Ceiling

_Auto-generated by ``benchmark_models.py compare``. Seasons in scope: [2024, 2025]._

## Verdict: PROMOTE `elite_head_plus_hybrid`

Phase 1 closed with the observation that MAE could be tightened (per_circuit / hybrid_blend dropped MAE 5.87 → 5.07) but **podium-hit and winner-hit were locked at the baseline values** (32.6% / 14.6%). Those variants re-rank using signals derived from the same `predicted_lap_time` — they don't add new information at the top of the field.

Phase 2 adds two binary classifier heads (podium + winner) trained on **elite-signal features** that the baseline pipeline doesn't expose: prior-only per-driver podium rate (last-5 + season), per-driver winner rate, per-driver per-circuit podium history, and a "qualifying dominance" gap signal. The heads then re-rank the top-6 of an anchor ordering. Both `elite_head` (baseline anchor) and `elite_head_plus_hybrid` (hybrid_blend anchor) clear the promotion gate with room to spare:

Gate (per the Phase-2 brief):
- Required: podium-hit ≥ +3pp OR winner-hit ≥ +2pp — **passed with +13.2pp and +6.2pp**.
- Forbidden: MAE regression >5% AND within-3 regression >5pp — **MAE improved 16%, within-3 improved 8pp**.
- Forbidden: calibration reliability degradation >20% — rank-derived calibration buckets are identical to baseline (re-rank only affects integer order, not the rank-derived P(win) used in the reliability surface).

### Why `elite_head_plus_hybrid` over `elite_head`

Both unlock the same podium/winner gains (the top-6 re-rank is identical because both use the same elite heads). `elite_head_plus_hybrid` additionally inherits hybrid_blend's MAE/within-3 wins on positions 7-22, so total MAE is 4.95 vs 5.03 and the per-season breakdown is consistently better. Promote the stacked variant.

## Features that moved the needle

Logistic-regression coefficients (normalized to sum to 1):

| feature | podium head | winner head |
|---|---:|---:|
| `driver_podium_rate_season` | **36.6%** | **28.4%** |
| `driver_podium_rate_5` | **24.0%** | **25.2%** |
| `qualifying_dominance` | 13.2% | 16.6% |
| `predicted_position` | 13.2% | 16.6% |
| `driver_winner_rate_season` | 9.4% | 9.8% |
| `driver_circuit_podium_rate` | 3.6% | 3.4% |

**Signal**: the prior-only season-level podium rate is doing 60-65% of the work. This is real elite-signal: it captures "Verstappen has been on the podium in 17/24 prior 2024 rounds — the baseline ordering missing him in the top-3 is wrong."

**Dead weight**: `driver_circuit_podium_rate` contributes <4%. With only 2024 seasons to draw on for early-2025 rounds, most drivers have ≤1 prior visit at most circuits and the feature is mostly NaN. Worth keeping for the cases where it's defined (e.g. Verstappen at Spa) but it's not load-bearing in 2024+2025. Will become more useful once 3+ historical seasons accumulate per circuit.

## Production wiring recipe (action items if promoted)

The benchmark proves the gate. To wire `elite_head_plus_hybrid` into the live pipeline:

1. **`export_round_data` in `export_website_data.py`** — after the L1 ensemble produces `predictedTime` and the round JSON is built, but before the probability layer runs:
   - Load `models/registry/elite_heads/podium_head.joblib` and `winner_head.joblib`.
   - Build elite features for the active round via `build_elite_features(history_df, season, round)` — pull `history_df` from `data/history.duckdb`.
   - Score `pod_probs` and `win_probs` for each driver.
   - Re-rank the top-6 of the current `classification[*].predictedPosition` ordering using `0.7 * P(podium) + 0.3 * P(win)`. (Helper available: `benchmark_models._rerank_top_n_by_elite`.)
   - Persist the elite probabilities into `classification[*]` as `elitePodiumProbability` and `eliteWinProbability` so the website can surface them.

2. **TypeScript contract** — extend `website/src/types/index.ts::ClassificationEntry` with the two new optional fields; mirror in `tests/test_website_data_schema.py`'s pydantic models.

3. **Phase-aware gating** — the heads need ≥5 prior rounds with actuals to train. Early in a season (R1-R5), pool across the prior season; below the threshold, fall back to the un-reranked ordering. The same `ELITE_HEAD_MIN_TRAIN_ROUNDS` guard from `benchmark_models.py` applies.

4. **Retraining cadence** — re-fit the heads after every round on the cumulative 2024 + 2025 (+ now in-season 2026) corpus. The fit takes <1s on 500-1000 rows; do it in the same step as Plackett-Luce sampling.

5. **Forward-eval extension** — add `precision@3` (podium) and `precision@1` (winner) to `forward_eval.py`'s per-round JSON so the next round's promotion decision has the right metrics.

6. **Backout switch** — gate the re-rank behind a `F1_USE_ELITE_HEADS` env var (default "1") so a regression detected in `forward_eval` can be rolled back instantly via the workflow YAML.

---

**Approximation disclosure** — the per-circuit and hybrid-blend variants run on a synth feature frame built from the DB rows (predicted_position + predicted_lap_time + archetype priors) rather than a full L1 retrain per round. The elite-head variants train classifiers on driver-level prior aggregates from the DB. See the APPROXIMATION NOTE blocks in `benchmark_models.py::predict_per_circuit` and `benchmark_models.py::predict_hybrid_blend` for the exact substitution. These are *signal-direction* benchmarks, not production wirings.
