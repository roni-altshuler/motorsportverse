# RaceIQ NASCAR website

Next.js static-export site for the MotorsportVerse NASCAR Cup Series project —
the same RaceIQ design system as the F1/F2/F3/Formula E sites, re-themed to the
NASCAR victory-lane yellow accent (`#FFD659`, registry palette). A LIGHT accent
on the black canvas, so the derived tokens flip: hovers darken and every
text-on-accent foreground (`--accent-ink`) is near-black.

## What's different from the Formula E parent clone

- **No doubleheaders** — every Cup round is its own venue/weekend; FE's
  weekend-pairing machinery is gone.
- **Track types** — every round is `superspeedway` / `intermediate` / `short` /
  `road`; badged on the calendar and race header, threaded through the
  narrative rules, and used as the four calibration strata on /accuracy.
- **Stage racing** — completed rounds carry `stageLaps` + per-stage top-10
  `stageResults`, rendered by `race-detail/StageResultsPanel`.
- **DNF hazard is first-class** — every prediction row carries `pDnf`; the
  race tables surface it as a subtle risk indicator and the narrative card
  flags high-hazard favourites.
- **The Chase** — /standings gains a sanctioned divergence: the playoff panel
  (`standings/ChasePanel`: top-16 cut line with the bubble highlighted,
  per-driver make-playoffs → title ladder, title-odds bars) fed by
  `playoff_projection.json`, plus a manufacturer standings panel. The Chase
  section ships ONLY when `historical_backtest/playoffs.json` reports
  `gate.pass === true`. Cumulative points projections are labelled "projection
  to end of regular season".

## Data

All pages read `public/data/nascar.json` (+ `rounds/`, `probabilities/`,
`forward_eval/`, `historical_backtest/`, `playoff_projection.json`,
`model_health.json`, `promotion_status.json`, `calibration_summary.json`,
`seasons.json`), generated by the Python pipeline:

```bash
# from the project root, after installing the NASCAR package + core/data:
python -m nascar_predictions.export   # writes website/public/data/nascar.json + fan-out
```

The TypeScript contract is `src/types/nascar.ts`; its pydantic mirror lives in
`../tests/test_website_data_schema.py` — change both together. Round
completed-ness keys on the export's finishing status, never on non-empty
result arrays (future rounds ship pre-seeded entry lists).

## Develop / build / verify

```bash
npm install
PAGES_BASE_PATH= npm run build              # static export → out/
node scripts/shoot.mjs [/tmp/nascar-shots]  # Playwright screenshot harness (after build)
```

`next.config.ts` reads `PAGES_BASE_PATH`; the unified Pages deploy sets it to
`/motorsportverse/projects/nascar`.

Race art (`src/lib/raceArt.ts`) must stay aerial venue photography — every URL
curl-verified; venues with no verified aerial fall back to the gradient card.
`src/components/{ui,magicui}` are byte-identical mirrors of the F1 canonical
set (`scripts/sync_shared_ui.mjs --check` at the repo root).
